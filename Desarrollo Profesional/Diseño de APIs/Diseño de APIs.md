# 🔌 Diseño de APIs

[[Desarrollo Profesional/Inicio Profesional|⬅️ Volver a Desarrollo Profesional]]

> [!abstract] Sobre esta sección
> Una API es un contrato: define cómo otros sistemas (y otros equipos) hablan con el tuyo. Un buen diseño de API es la diferencia entre una integración que se entiende sola y una fuente perpetua de tickets de soporte. Esta sección cubre el diseño de APIs desde los principios —REST y su modelo de madurez, el diseño de recursos, el versionado, la paginación, los errores— hasta las alternativas (gRPC para comunicación interna de alto rendimiento, GraphQL para clientes con necesidades de datos flexibles) y cuándo elegir cada una.
